# APP全模块开发文档（含8大模块·宠物模块优化+其余7大模块补充）

# 全局通用规则（贯穿8大模块，统一规范，确保兼容性、可落地性）

## 1. 技术栈固定
- **前端**：Flutter（统一跨平台开发框架）
- **后端**：Node.js（统一服务端运行环境）
- **数据库**：SQLite（统一本地数据存储）
- **ORM框架**：Sequelize（统一数据库交互管理）
- **核心要求**：所有模块无新增无关框架/依赖，确保技术栈一致性和可维护性

## 2. 视觉规范
### 2.1 颜色体系（完全区分，无重叠）
- **背景颜色**：#F5F7FA（主背景）、#FFFFFF（卡片背景）
- **按钮颜色**：#367BF5（主按钮）、#FF6B35（次按钮）、#409EFF（文字按钮）、#C0C4CC（禁用按钮）
- **字体颜色**：#333333（主字体）、#666666（副字体）、#909399（辅助字体）
### 2.2 设计原则
- 视觉清晰、层级分明
- 所有模块视觉风格统一
- 适配移动端单手握持操作

## 3. 版本适配（双版本兼容）
### 3.1 基础版
- **存储方式**：本地存储所有核心数据（用户、任务、宠物等）
- **功能特点**：离线可用，数据本地安全
- **适用场景**：无网络环境或临时使用

### 3.2 优化版
- **存储方式**：云端+本地双重存储
- **功能特点**：数据实时同步，多设备互通
- **异常处理**：网络异常时本地数据兜底，确保功能可用

## 4. 逻辑联动（8大模块深度联动）
### 4.1 核心联动链
**用户模块 → 宠物模块 → 评估系统 → 任务模块 → 激励模块 → 数据统计模块 → 社交互动模块 → 个人中心与设置模块**

### 4.2 联动原则
- 数据流转顺畅，无逻辑冲突
- 模块间接口统一，调用规范
- 核心数据共享，避免数据冗余

## 5. 开发要求
- **前后端分工明确**：前端负责页面展示、交互逻辑、本地存储；后端负责接口开发、业务逻辑、云端存储
- **详细拆解**：每个模块需拆解前后端实现、逻辑校验、数据流转、异常处理
- **步骤明确**：开发流程清晰，逻辑严谨，AI可直接理解并输出对应代码/方案
- **重点强化**：宠物、任务、评估等核心模块需强化细节，确保落地性

## 6. 交互要求
- **操作流畅**：所有页面操作无卡顿，响应及时
- **反馈机制**：点击动效、toast提示、加载状态等反馈标准化
- **文案规范**：简洁明了、不晦涩，符合用户认知
- **适配设计**：移动端单手握持操作，避免误触、冗余交互

## 7. 数据规范
- **字段量化**：所有数据表字段量化、可配置，无主观模糊字段
- **存储原则**：遵循"最小必要"原则，避免数据冗余
- **安全要求**：敏感数据（优化版用户信息）加密存储
- **日志记录**：完整的操作日志，便于后续问题排查

## 8. 接口规范
- **统一返回格式**：`{code: 状态码, data: 响应数据, msg: 提示信息}`
- **状态码定义**：200=成功，400=参数错误，401=令牌无效，500=服务器错误
- **版本标识**：所有接口需携带`versionType`参数（0=基础版，1=优化版）
- **令牌校验**：优化版接口需进行token令牌校验，确保接口安全

# 模块1：用户核心模块（基础核心，支撑所有模块）

## 1. 功能核心定义

### 1.1 核心功能
- **用户注册**：优化版支持手机号注册，基础版无需注册
- **用户登录**：双版本适配，基础版默认游客登录，优化版手机号密码登录
- **个人信息管理**：支持昵称、头像等基础信息修改
- **权限控制**：基于Token的权限校验（优化版）

### 1.2 版本差异
- **基础版**：默认游客账号（userId=0），本地存储，无需注册流程
- **优化版**：手机号注册登录，云端+本地双重存储，多设备同步

### 1.3 核心价值
作为APP所有功能的基础入口，用户信息与宠物、任务、评估等数据深度关联，确保用户个性化体验。

## 2. 技术实现核心要求

### 2.1 双版本适配
- 基础版：本地存储所有核心数据，退出APP不清除数据（除非手动清除）
- 优化版：云端+本地双重存储，数据实时同步，异常时本地数据兜底

### 2.2 安全要求
- 密码加密：采用bcrypt加密算法，不存储明文密码
- Token校验：优化版登录生成Token，有效期7天，过期自动刷新
- 数据安全：敏感信息加密存储，操作日志完整记录

### 2.3 模块联动
- 用户ID作为核心关联字段，关联宠物、任务、评估、统计等所有数据表
- 确保数据归属正确，流转顺畅，无逻辑冲突

### 2.4 异常处理
- 登录失败、注册失败、信息修改失败等场景，返回清晰错误提示
- 支持重试机制，确保用户操作体验

### 3.2 后端（Node.js）开发：核心接口实现

#### 3.2.1 接口规范
- **统一返回格式**：`{code: 状态码, data: 响应数据, msg: 提示信息}`
- **状态码定义**：200=成功，400=参数错误，401=令牌无效，500=服务器错误
- **版本标识**：所有接口需携带`versionType`参数

#### 3.2.2 核心接口

##### （1）注册接口（仅优化版）
- **接口地址**：POST /api/user/register
- **接收参数**：phone、password、nickname（可选）、versionType=1
- **核心逻辑**：
  1. 参数校验：手机号格式、密码长度≥6位
  2. 唯一性校验：手机号是否已注册
  3. 密码加密：bcrypt加密
  4. 数据插入：创建用户信息
- **返回数据**：`{userId: 新增用户ID, phone: 手机号, nickname: 昵称}`

##### （2）登录接口（双版本通用）
- **接口地址**：POST /api/user/login
- **接收参数**：
  - 基础版：userId=0，versionType=0
  - 优化版：phone、password、versionType=1
- **核心逻辑**：
  1. 版本适配：基础版直接创建/查询游客账号
  2. 优化版：校验手机号密码，生成Token
  3. 更新登录时间
- **返回数据**：`{userId: 用户ID, nickname: 昵称, avatar: 头像, token: Token（优化版）, versionType: 版本类型}`

##### （3）退出接口（仅优化版）
- **接口地址**：POST /api/user/logout
- **接收参数**：userId、token、versionType=1
- **核心逻辑**：校验Token，清空Token
- **返回数据**：`{}`

##### （4）密码找回接口（仅优化版）
- **接口地址**：POST /api/user/findPassword
- **接收参数**：phone、newPassword、versionType=1
- **核心逻辑**：校验手机号，更新加密密码
- **返回数据**：`{}`

##### （5）个人信息查询接口（双版本通用）
- **接口地址**：GET /api/user/info
- **接收参数**：userId、versionType、token（优化版）
- **核心逻辑**：校验参数/Token，查询用户信息
- **返回数据**：用户信息对象（不含密码）

##### （6）个人信息修改接口（双版本通用）
- **接口地址**：POST /api/user/update
- **接收参数**：userId、versionType、token（优化版）、nickname（可选）、avatar（可选）
- **核心逻辑**：校验参数/Token，更新非空字段
- **返回数据**：更新后的用户信息（nickname、avatar）

# 模块2：任务核心模块（核心功能，联动宠物、评估模块）

## 1. 功能核心定义

### 1.1 核心功能
- **任务发布**：系统智能生成适配用户能力的任务
- **任务接收**：用户接收推荐的任务，开始执行
- **任务完成**：用户提交任务完成情况，获取收益
- **任务推荐**：基于评估等级、宠物状态、心流值的精准推荐
- **任务管理**：任务列表查看、筛选、搜索等管理功能

### 1.2 任务类型与难度
- **任务类型**：light（轻量）、fun（趣味）、skill（技能）、normal（常规）
- **任务难度**：1-5星（1星最简单，5星最难）

### 1.3 核心价值
作为APP的核心功能模块，联动评估系统、宠物模块、激励模块，贴合心流理论，确保任务难度与用户能力匹配，提升用户粘性和成就感。

## 2. 技术实现核心要求

### 2.1 双版本适配
- **基础版**：本地生成、存储任务数据，确保离线可用
- **优化版**：云端发布、同步任务数据，实时更新，异常时本地兜底
- **一致性**：双版本任务逻辑完全一致，确保用户体验统一

### 2.2 推荐逻辑
- **优先级**：评估等级→宠物状态→心流值
- **评估等级匹配**：严格对应任务难度区间，确保任务与用户能力适配
- **宠物状态反哺**：根据宠物状态推荐对应类型任务，促进宠物全面发展
- **心流值调节**：根据用户心流状态微调任务难度，保持最佳挑战感

### 2.3 实时性要求
- 任务数据实时刷新，状态变更及时反馈
- 任务完成后立即更新：任务状态、宠物状态、激励数据、统计数据
- 确保数据一致性，避免数据冲突

### 2.4 操作便捷性
- 支持任务筛选：按类型、难度
- 支持任务搜索：按任务名称
- 操作流程简洁，反馈及时

### 2.5 异常处理
- 任务发布、接收、完成等场景的错误提示清晰
- 支持重试机制，确保数据一致性
- 网络异常时本地数据兜底，恢复后自动同步

### 3.2 后端（Node.js）开发：核心接口实现

#### 3.2.1 接口规范
- **统一返回格式**：`{code: 状态码, data: 响应数据, msg: 提示信息}`
- **状态码定义**：200=成功，400=参数错误，401=令牌无效，500=服务器错误
- **版本标识**：所有接口需携带`versionType`参数

#### 3.2.2 核心接口

##### （1）任务推荐接口（双版本通用）
- **接口地址**：GET /api/task/recommend
- **接收参数**：userId、petId、versionType、token（优化版）
- **核心逻辑**：
  1. 评估等级匹配（最高优先级）：根据用户评估等级匹配任务难度区间
  2. 宠物状态反哺：根据宠物状态推荐对应类型任务
  3. 心流值调节：根据心流状态微调任务难度
  4. 推荐任务筛选：每次推荐3-5个未过期、未接收的任务
- **返回数据**：`{taskList: 推荐任务列表, abilityAdaptReason: "推荐原因"}`

##### （2）任务接收接口（双版本通用）
- **接口地址**：POST /api/task/receive
- **接收参数**：userId、taskId、petId、versionType、token（优化版）
- **核心逻辑**：
  1. 参数校验：任务未过期、未完成、未接收
  2. 数据更新：更新任务状态为已接收
- **返回数据**：`{taskId: 任务ID, taskName: 任务名称, receiveTime: 当前时间}`

##### （3）任务完成接口（双版本通用）
- **接口地址**：POST /api/task/finish

# 模块5：激励核心模块（优化版·多维度激励+养成交环）

## 1. 功能核心定义

### 1.1 核心功能
- **激励触发**：基于模块3评估等级和模块4任务完成数据，自动触发对应等级的激励
- **收益发放**：整合积分、宝箱、成就、周期福利四大激励形式，实时发放奖励
- **权益解锁**：根据评估等级和任务完成情况，解锁不同等级的激励权益
- **数据统计**：实时统计激励获取、消耗、过期等数据，生成详细报表

### 1.2 版本差异
- **基础版**：本地存储激励数据，计算收益加成，确保离线可用
- **优化版**：云端+本地双重存储，数据实时同步，异常时本地兜底

### 1.3 核心价值
作为APP用户粘性核心板块，搭建"评估-任务-激励-宠物"多维度闭环激励体系，通过等级绑定、任务驱动、养成交联的激励形式，提升用户任务参与度，强化与模块3/4的功能联动。

## 2. 技术实现核心要求

### 2.1 双版本适配
- **基础版**：本地存储所有激励数据，退出APP不清除数据（除非手动清除）
- **优化版**：云端+本地双重存储，数据实时同步，异常时本地数据兜底

### 2.2 核心激励体系
- **四大激励形式**：积分（可兑换）、宝箱（随机掉落）、成就（里程碑奖励）、周期福利（每日/每周/月度）
- **收益加成规则**：评估等级基础收益（S→1.5、A→1.4、B→1.2、C→1.0、D→0.8）+ 任务质量加成（≥80分→1.2倍，≥90分→1.5倍）
- **个性化适配**：记忆用户激励偏好，联动模块4任务推荐系统

### 2.3 模块联动
- **与模块3联动**：评估等级决定激励基础收益，激励收益反哺宠物成长
- **与模块4联动**：任务完成触发激励，高质量任务额外加成，激励偏好影响任务推荐

### 2.4 异常处理
- 网络异常时自动切换至本地模式，确保基本功能可用
- 操作错误时提供清晰的错误提示，支持重试机制

## 3. 核心功能开发与前后端分工
### （一）核心新增：评估-任务双维度激励绑定体系
1.  评估等级绑定核心权益：S级解锁精英宝箱、专属成就、月度稀有福利，A级解锁进阶宝箱、高阶积分兑换项，B/C/D级依次递减，等级未达标时高阶激励入口置灰并明确解锁条件；
2.  任务数据触发激励解锁：完成模块4每日5个适配任务解锁每日宝箱，每周20个适配任务解锁周福利，连续7天高质量完成（效果分≥80）任务解锁成就，任务完成质量决定激励加成比例；
3.  收益加成叠加规则：激励基础收益（由评估等级决定）+ 任务质量加成（效果分≥80享1.2倍，≥90享1.5倍），两者可叠加计算，如S级评估+90分任务效果分，总收益系数=1.5×1.5=2.25倍；
4.  权益实时联动：模块3评估等级提升、模块4任务完成数据达标后，激励模块对应权益/宝箱/福利自动解锁，无需手动触发，解锁后实时推送提醒。

### （二）四大激励形式整合与优化
1.  积分体系：积分获取与任务完成、成就解锁、签到绑定，可兑换宠物成长道具、福利，设置30天有效期，临近过期实时提醒，支持明细查看、本地导出；
2.  宝箱体系：分基础/进阶/精英/专属四级，对应不同评估等级，解锁后开启随机掉落奖励，等级越高稀有道具掉落概率、基础收益越高，累计开箱可解锁额外福利；
3.  成就体系：按“任务完成、宠物成长、评估提升、连续打卡”分类，解锁后自动领取奖励，点亮全部分类成就可解锁专属宠物形象/永久收益加成；
4.  周期福利：包含每日签到/每日任务、每周任务、月度等级福利，奖励随连续参与/任务完成量递增，支持补做任务补发未解锁福利（每月限3次）。

### （三）激励与宠物成长深度闭环（反哺模块3）

### （四）激励偏好与任务推荐联动（反哺模块4）
1.  自动记忆激励偏好：系统记录用户积分兑换、宝箱开启的偏好（如偏好兑换营养道具、优先开启精英宝箱）；

### （一）数据库设计（新增+修改表，适配社交+联动模块3-5）
基于原有SQLite数据库，**新增`social`核心表**存储社交基础数据，修改`incentive`/`statistics`表新增社交关联字段，所有字段量化可配置，确保与模块3-5数据互通。
1.  **social表**（新增，双版本，社交核心表）：存储好友关系、互动记录、分享记录、排行榜数据

    - `shareRecord`：成果分享记录（JSON格式）；`rankScore`：排行榜综合得分；`rankPrivacy`：社交隐私设置（JSON格式）；

1.  **社交核心数据接口（新增，双版本通用）**
    接口地址：GET /api/social/core
    接收参数：userId、petId、versionType、token（优化版必填）
    核心逻辑：① 参数/令牌校验；② 从social表查询好友列表、互动次数、分享记录、隐私设置；③ 结合模块3-5数据，生成用户社交展示数据；④ 校验今日互动次数是否超限；⑤ 返回核心社交数据+可互动标识。
    返回数据：code=200，data={friendList: 好友列表, interactInfo: 互动信息, shareRecord: 分享记录, privacySet: 隐私设置, canInteract: 可互动标识}，msg="社交核心数据查询成功"。

2.  **好友互动接口（新增，双版本通用，含2个子接口）**
    - 好友点赞：POST /api/social/friend/like → 校验今日点赞次数是否超限，更新用户点赞数、好友被点赞数，给双方发放额外宠物经验，联动模块3宠物状态更新接口；
    - 好友助力：POST /api/social/friend/help → 校验今日助力次数是否超限，更新用户助力数、好友被助力数，给双方发放额外积分（模块5），联动模块5激励核心数据接口。

3.  **成果分享接口（新增，双版本通用）**
    接口地址：POST /api/social/share
    接收参数：userId、petId、shareType（成果类型）、shareContent（成果内容）、versionType、token（优化版必填）
    核心逻辑：① 校验成果是否为可分享类型；② 生成标准化分享卡片，更新用户分享记录；③ 给分享者发放基础分享奖励（积分），联动模块5激励核心数据接口；④ 返回分享卡片链接/二维码。
    返回数据：code=200，data={shareCard: 分享卡片数据, shareLink: 分享链接, shareQr: 二维码Base64, reward: 分享奖励}，msg="成果分享成功"。

4.  **排行榜接口（新增，双版本通用，含2个子接口）**
    - 排行榜查询：GET /api/social/rank → 按评估等级/任务完成数/宠物等级查询排行榜数据，基础版返回本地排行榜，优化版返回云端排行榜，展示前100名+用户自身排名；
    - 排行榜奖励领取：POST /api/social/rank/receive → 校验用户昨日排行榜排名，发放对应社交福利（宝箱/积分），联动模块5激励操作接口，更新社交福利领取记录。

5.  **社交设置与联动接口（新增，双版本通用）**
    - 隐私设置：POST /api/social/set/privacy → 保存用户社交隐私设置，更新social表隐私设置字段，实时生效；
    - 社交与激励联动：POST /api/social/link/incentive → 社交行为（点赞/助力/分享/上榜）触发后自动调用，计算社交激励收益，联动模块5激励核心数据接口，更新激励数据。

## 四、代码实现要点
1. **社交数据实时同步机制**：前端使用Stream监听模块3-5数据变化，宠物升级/任务完成/激励获取时自动刷新社交页面数据；后端使用WebSocket推送实时社交动态（好友点赞/助力/分享/排行榜更新），确保数据一致性。
2. **互动次数精确控制**：后端使用Redis缓存今日互动次数，每次互动前校验次数是否超限，超限则返回错误码；前端本地缓存剩余次数，每次互动后立即更新UI，避免重复请求。
3. **排行榜实时计算**：后端定时任务（每5分钟）计算排行榜数据，缓存至Redis；前端首次加载排行榜时从缓存读取，后续通过WebSocket推送增量更新，减少数据库查询压力。
4. **分享卡片生成优化**：前端使用Canvas绘制标准化分享卡片，支持自定义背景；后端生成分享链接，链接包含分享ID和加密参数，防止伪造分享链接刷奖励。
5. **隐私设置实时生效**：前端使用Provider管理隐私设置状态，设置修改后立即更新所有社交页面UI；后端隐私设置字段存储为JSON格式，查询时动态过滤隐私数据。

## 六、核心业务流程

2. **成果分享流程**：用户在模块3/5完成成果→成果图标显示"可分享"标识→点击标识跳转分享页→生成标准化分享卡片→选择分享渠道→分享成功→显示分享奖励→好友点赞后分享者获得额外奖励
3. **排行榜竞争流程**：用户完成模块3-5任务→排行榜数据实时更新→用户查看排行榜→查看自身排名及差距→点击"超越TA"跳转任务页→完成任务提升排名→每日早8点发放排行榜奖励
4. **隐私设置流程**：用户进入社交设置页→勾选/取消勾选隐私选项→点击保存→后端更新隐私设置→前端实时刷新所有社交页面→隐私数据立即隐藏/显示

## 七、模块联动关系
1. **与模块3（宠物成长）联动**：
   - 好友点赞/助力获取额外宠物经验，加速宠物升级
   - 宠物等级提升/技能解锁可分享成果，展示社交成就
   - 排行榜按宠物等级排名，推动用户养成宠物
2. **与模块4（任务完成）联动**：
   - 高难度任务完成可分享成果，展示任务成就
   - 排行榜按任务完成数排名，推动用户完成任务
   - "超越TA"功能跳转任务页，反哺任务参与度
3. **与模块5（激励获取）联动**：
   - 好友助力/分享点赞/排行榜上榜获取额外积分/宝箱
   - 排行榜奖励发放至激励模块，宝箱可直接开启
   - 社交积分可兑换宠物成长道具，强化养成交联
4. **与模块7（个人中心）联动**：
   - 个人中心展示社交数据（好友数/社交互动数）
   - 社交待办事项聚合至个人中心待办列表
   - 社交设置与个人中心全局设置联动

## 八、总结与亮点回顾
1. **轻量化社交体系**：仅保留与宠物养成、任务完成相关的核心社交功能，拒绝无意义社交，社交流程简化，一键操作，降低用户学习成本。

## 三、技术实现核心要求（前后端分工）
前端负责个人中心卡片式布局展示、数据可视化渲染、全量设置项交互、本地配置存储/数据导出，以及与模块3-6的快捷跳转联动；后端负责个人核心数据聚合接口、配置云端同步接口、数据报表生成接口、账号安全接口，所有接口通过`versionType`区分版本，优化版需token令牌校验，接口返回格式统一为`{code: 状态码, data: 响应数据, msg: 提示信息}`。

### （一）数据库设计（新增+修改表，适配数据聚合+配置）
基于原有SQLite数据库，**新增`user_center`核心表**存储个人配置、数据聚合标识，修改`statistics`表新增数据趋势统计字段，所有字段量化可配置，确保与模块3-6数据互通。
1.  **user_center表**（新增，双版本，个人中心核心表）：存储个人配置、常用入口、待办事项、账号信息
    - `userId`：用户ID（关联主表）；`petId`：宠物ID（联动模块3）；`avatar`：用户头像（Base64/链接）；`nickname`：用户昵称；
    - `commonEntry`：常用入口列表（JSON格式，关联模块3-6）；`toDoList`：待办事项（JSON格式，含模块、内容、状态）；
    - `interactSet`：交互设置（JSON格式，按键反馈、震动强度等）；`privacySet`：全局隐私设置（JSON格式，覆盖模块6）；
    - `remindSet`：全局提醒设置（JSON格式，覆盖模块3-6）；`dataSet`：数据设置（JSON格式，同步频率、导出格式等）；
    - `updateTime`：配置更新时间；`syncStatus`：云端同步状态（0=未同步，1=同步中，2=已同步）。
2.  **statistics表**（修改，通用统计表）：新增数据趋势统计字段
    - `weekPetGrow`：本周宠物成长值数组（JSON）；`weekTaskFinish`：本周任务完成数数组（JSON）；`weekIntegralGet`：本周积分获取数数组（JSON）；
    - `dataExportRecord`：数据导出记录（JSON格式，含模块、时间、格式）。

### （二）后端（Node.js）开发：核心接口实现（新增+联动，对接模块3-6）
所有接口适配双版本，深度联动模块3-6核心接口，实现数据聚合、配置同步、报表生成，配置修改后实时推送至对应模块。
1.  **个人核心数据聚合接口（新增，双版本通用）**
    接口地址：GET /api/user/core
    接收参数：userId、petId、versionType、token（优化版必填）
    核心逻辑：① 参数/令牌校验；② 聚合模块3-6核心数据（评估等级、宠物状态、任务统计、激励收益、社交数据）；③ 从user_center表获取待办事项、常用入口；④ 从statistics表获取本周数据趋势；⑤ 返回聚合数据+可视化配置。
    返回数据：code=200，data={coreCard: 核心数据卡片, trendData: 本周趋势数据, toDoList: 待办事项, commonEntry: 常用入口}，msg="个人核心数据查询成功"。
2.  **个性化配置接口（新增，双版本通用，含保存/查询子接口）**
    - 配置查询：GET /api/user/set/query → 从user_center表查询所有配置项，返回至设置页；
    - 配置保存：POST /api/user/set/save → 接收用户配置修改参数，更新user_center表，**实时同步至对应模块**，返回“设置已生效”标识。
3.  **数据报表导出接口（新增，双版本通用）**
    接口地址：POST /api/user/data/export
    接收参数：userId、moduleType（模块3-6）、timeType、fileFormat、versionType、token（优化版必填）
    核心逻辑：① 校验参数；② 按模块/时间统计数据；③ 生成Excel/CSV格式报表；④ 基础版返回文件二进制流（本地保存），优化版返回文件链接+云端备份；⑤ 记录导出记录至statistics表。
4.  **账号安全接口（新增，双版本通用）**
    - 信息修改：POST /api/user/info/edit → 修改昵称/头像/签名，同步至所有模块展示；
    - 密码修改：POST /api/user/pwd/change → 验证旧密码，更新新密码，返回修改成功；
    - 设备管理（优化版）：GET /api/user/device/list → 查看登录设备；POST /api/user/device/logout → 下线异常设备。

### （三）前端（Flutter）开发：核心交互逻辑（新增+优化，适配单手握持+联动模块3-6）
#### 2. 核心交互逻辑（新增+优化，适配按键优化+评估系统）
##### （1）按键交互优化（核心）
- 所有个人中心/设置相关按键（数据卡片、快捷入口、配置开关、确认/保存键、导出/同步键）均适配移动端单手握持操作，**核心操作按键热区≥44×44px**，常用入口/配置开关等高频按键热区放大至≥50×50px，数据卡片整卡可点击（热区覆盖全卡），避免指尖点空；按键间距按功能分区，核心操作按键间距≥20px，次要设置项按键间距≥15px，小屏（＜375px）所有按键横向居中，大屏（≥375px）均匀分布在页面安全区内，贴合单手握持操作范围。
- 按键视觉与状态分级：① 确认/保存/导出等主操作按键，复用全局主按钮样式（#367BF5），尺寸为页面宽度80%/高度48px，文字16sp，不可操作时（如无数据可导出）置灰（#C0C4CC）+ 透明度60%，添加悬浮提示（如“暂无本周数据可导出”）；② 配置开关采用全局标准化开关样式，开启为绿色（#67C23A）、关闭为灰色，开关右侧标注配置说明（如“按键震动：开启”），关联多模块的配置项标注关联模块（如“全局小红点提醒（关联所有模块）”）；③ 快捷入口采用“图标+文字”卡片式设计，选中/常用入口添加浅蓝底色（#E6F7FF）标识，未选中为白色底色，入口图标根据模块属性设计（如宠物模块为宠物图标、任务模块为任务图标），视觉统一。

- 状态联动的按键权限：按键状态与模块3-6数据、用户配置实时联动，① 若模块5有未领取激励，“激励”快捷入口/数据卡片右上角添加小红点数字提醒，领取后红点实时消失；② 若用户在隐私设置中隐藏评估等级，个人中心评估等级卡片将模糊展示，点击可解锁查看（需验证），同时社交模块同步隐藏；③ 若优化版云端同步中，“同步”按键置灰并显示加载动画，同步完成后自动恢复为“同步成功”状态，触发横幅提示，按键状态变化无延迟，实时响应数据更新。

##### （2）核心数据聚合与快捷跳转交互逻辑（新增，核心衔接模块3-6）
- 数据实时刷新与可视化：个人中心核心数据卡片、趋势图表**每秒自动刷新一次**，模块3-6数据变化后（如宠物升级、完成任务、获取积分），个人中心数据同步更新，更新时触发**数字渐变动画**（如积分从100变为150，数字平滑过渡），趋势图表实时重绘，无需手动下拉刷新；点击趋势图表右上角“筛选”按键，可按模块（宠物/任务/激励）筛选查看趋势，筛选后图表无延迟切换，同时显示对应模块的趋势说明（如“本周宠物成长值最高为周三：500点”）。
- 待办事项聚合与一键处理：个人中心顶部设置“待办事项”入口，入口右上角以红色数字标注待办总数，点击入口弹出待办列表弹窗（底部弹出式，贴合单手握持），列表中每条待办标注**所属模块、待办内容、处理入口**（如“激励模块：3个宝箱未开启 → 立即领取”），点击“立即处理”一键跳转至对应模块待办页面，处理完成后待办列表实时刷新，总数自动递减，待办全部完成后入口红点消失，显示“暂无待办事项”。
- 常用入口自定义与置顶：点击个人中心“自定义常用”按键，弹出所有模块入口列表（模块3-6），支持**勾选+拖拽排序**，最多勾选6个，勾选后点击“保存”，常用入口立即置顶至个人中心核心区，按拖拽顺序排列；常用入口卡片添加浅蓝底色标识，点击可一键直达对应模块核心页面，跳转后保留原页面操作状态（如任务模块的筛选条件、激励模块的积分兑换分类），返回个人中心时无需重新加载。
- 数据卡片直达详情：5大核心数据卡片（评估等级、宠物状态、任务统计、激励收益、社交互动）均支持**整卡点击跳转**，① 评估等级卡片点击直达模块3评估等级详情页，展示等级提升条件、当前进度；② 宠物状态卡片点击直达模块3宠物主页，展示宠物成长值、技能、道具；③ 其余卡片依次跳转至对应模块核心统计页，卡片右下角添加“→”箭头标识，提示“点击查看详情”，降低用户学习成本。

##### （3）个性化配置一键生效交互逻辑（新增，覆盖模块3-6）
- 配置项分类与极简操作：设置页按“交互设置、隐私设置、提醒设置、数据设置、账号设置”五大类分区，每类设置项折叠展示，点击分类标题展开/折叠，折叠后显示当前核心配置状态（如“交互设置：按键震动开启、光效开启”）；所有配置项均为**一键操作**（开关/选择项/滑块），无冗余输入步骤，如按键震动强度通过滑块调节（弱/中/强），滑动后实时预览震动效果，无需保存，实时生效。
- 配置修改实时同步与生效：所有配置项修改后**无需手动点击保存**（开关/滑块类），或点击一次“保存”按键（信息修改/多选项类），立即同步至`user_center`表，并**实时推送至对应模块**，覆盖模块原有配置，无需重启APP；如修改“按键震动强度”为“弱”，则模块3-6所有按键的震动强度立即变为弱，修改“社交隐私：隐藏评估等级”，则模块6社交排行榜、好友列表中立即隐藏评估等级，同步生效后触发“设置已生效（关联XX模块）”的横幅提示（2s自动消失）。
- 配置恢复与默认值：设置页每类配置项底部添加“恢复默认”按键，点击后弹出二次确认弹窗（“确认恢复XX设置为默认值？”），确认后该类所有配置项立即恢复为系统默认值，同步至所有关联模块，并触发“已恢复默认设置”提示；默认值可根据用户使用习惯自动适配（如高频使用社交模块，默认开启社交提醒）。
- 配置预览与说明：所有配置项均添加**详细说明提示**，长按配置项/点击右侧“？”图标，弹出配置说明弹窗，说明包含“配置作用、关联模块、操作效果”，如长按“按键光效”配置项，弹窗显示“开启后，所有模块核心按键点击成功将触发光效动效，提升交互感知，关联模块3-6所有按键”；部分配置项支持实时预览（如震动强度、光效样式），修改后立即预览效果，提升配置体验。

##### （4）数据管理与导出交互逻辑（新增，适配双版本）
- 多维度数据筛选与导出：点击“数据导出”入口，进入导出页面，支持按**模块（任务/激励/社交/全部）、时间（今日/本周/本月/自定义）、文件格式（Excel/CSV）** 筛选，筛选条件通过下拉选择框选择，自定义时间支持选择开始/结束日期（日历选择器，适配单手握持）；所有筛选条件选择完成后，“导出”按键激活，点击后触发“正在导出，请稍候”的加载提示（按钮内小型加载动画），导出完成后，基础版弹出“保存至本地”提示，选择保存路径后完成导出，优化版同时弹出“本地保存+云端备份”选项，支持一键双保存。
- 导出记录与查看：导出完成后，自动生成导出记录，在“数据管理”页展示，记录包含**导出模块、时间、格式、保存路径/云端链接**，记录按时间倒序排列，点击记录可直接查看/打开导出文件，基础版记录保留本地，优化版记录云端同步，跨设备可查看；记录支持手动删除，删除前弹出二次确认，避免误删。
- 本地缓存清理与数据删除：点击“清理缓存”按键，弹出确认弹窗（“确认清理本地缓存？将释放X MB存储空间”），确认后开始清理，清理完成后显示“缓存清理完成，释放X MB空间”；点击“删除历史记录”（如社交互动记录、积分明细），支持按时间筛选删除（如删除7天前记录），删除前提示“删除后数据不可恢复”，确认后实时删除并刷新对应模块数据，无残留。
- 双版本数据同步交互：① 基础版：仅支持本地数据存储，数据管理页显示“本地数据已保存”，无同步按键；② 优化版：支持**手动/自动同步**，手动同步点击“立即同步”按键，触发同步加载动画，同步完成后显示“同步成功，云端数据已更新”，同步失败显示“同步失败，点击重试”；自动同步可在“数据设置”中选择同步频率（每日凌晨/每周日/每次退出APP），开启后显示“自动同步已开启，下次同步时间：X年X月X日”，同步状态实时展示在个人中心右上角。

##### （5）账号安全与跨模块联动交互逻辑（新增，覆盖模块3-6）
- 个人信息修改与实时同步：点击“个人信息”入口，进入信息修改页，支持修改昵称、头像、个性签名，头像支持“相机拍摄/相册选择”，选择后实时预览，点击“保存”按键，信息立即更新至user_center表，并**同步至所有模块展示**（如社交排行榜、好友列表、成果分享卡片、个人中心数据卡片），同步完成后所有模块的个人信息无延迟刷新，无需手动重启；修改昵称/头像后，触发“信息修改成功，已同步至所有模块”的横幅提示。
- 密码修改与设备管理：修改密码时，需依次输入“旧密码、新密码、确认新密码”，输入过程中实时校验（如新密码长度≥6位、新旧密码不一致），校验不通过时实时提示（如“新密码长度不足6位”），校验通过后点击“确认修改”，密码立即更新，同时下线所有登录设备，触发“密码修改成功，请重新登录”的提示；优化版设备管理页，展示所有登录设备（设备名称、系统、登录时间、登录地点），异常设备（如陌生设备）标红显示，点击“下线”按键，弹出确认弹窗，确认后该设备立即下线，无法再次登录，触发“异常设备已下线，账号安全已保护”的提示。
- 跨模块配置联动与隐私保护：个人中心的**全局配置项**修改后，实时同步至模块3-6，实现“一次设置，全模块生效”，① 交互设置（如按键震动、光效）：同步至模块3-6所有按键，震动强度/光效样式统一；② 隐私设置（如隐藏评估等级、隐藏宠物成长值）：同步至模块6社交模块（排行榜、好友列表）+ 个人中心数据卡片，统一隐藏；③ 提醒设置（如全局小红点、横幅提醒）：同步至模块3-6所有提醒入口，提醒方式统一；配置联动无延迟，修改后立即生效，无需在各模块重复设置。
- 退出/注销与数据保护：点击“退出账号”，弹出确认弹窗（“确认退出账号？退出后将保留本地配置数据”），确认后退出当前账号，返回登录页，本地配置/数据保留（基础版/优化版均保留）；点击“注销账号”，需依次完成“密码验证、阅读注销须知、勾选确认注销”，注销须知明确提示“注销后本地数据立即清除，云端数据保留30天，30天后永久删除，不可恢复”，确认注销后，立即清除本地所有数据，下线所有登录设备，触发“账号注销成功，感谢使用”的提示，注销流程不可逆，保障数据安全。

##### （6）全局异常处理与通用交互逻辑（优化，统一全模块体验）
- 网络异常交互：网络断开时，个人中心/设置页自动切换至本地模式，顶部显示黄色提示栏（“网络异常，仅可查看/修改本地配置，无法导出/同步云端数据”），本地可正常查看个人数据、修改本地配置、查看本地导出记录，无法进行云端同步、在线导出、账号设备管理等操作；网络恢复后，提示栏自动消失，异步同步本地未同步的配置/数据至云端，同步完成后刷新页面，补充显示云端数据。
- 操作错误与异常提示：① 无数据可导出时，点击“导出”按键触发toast提示（“暂无所选时间/模块的有效数据”）；② 密码修改时旧密码错误，触发toast提示（“旧密码错误，请重新输入”）；③ 清理缓存/删除数据时，未勾选确认，触发提示（“请勾选确认后再操作”）；④ 服务器错误导致操作失败（如保存配置、导出数据），弹出弹窗提示（“操作失败，请点击重试”），含“重试/取消”按键，点击重试可重新调用接口，不影响其他操作。
- 加载交互标准化：页面级操作（进入个人中心、数据导出页、设备管理页）显示**全屏轻量加载**（含APPlogo+“加载中...”文案），按钮级操作（保存配置、导出数据、同步云端、修改密码）显示**按钮内小型加载动画**（不改变按钮尺寸/位置），无重复加载、过度加载，加载完成后平滑渲染内容，无页面闪烁、卡顿；数据趋势图表加载时显示“图表加载中...”，加载完成后实时重绘。
- 操作记忆与个性化：系统自动记忆用户的**操作偏好**，如个人中心的趋势图表默认筛选模块、数据导出的默认格式/时间、设置页的默认展开分类，退出APP后重新进入时，恢复至上次操作状态，无需重复设置；偏好数据存储在user_center表，基础版本地保存，优化版云端同步，跨设备保持一致操作体验；同时支持“恢复所有操作记忆”，一键重置为系统默认偏好。
- 空数据与异常状态处理：当某模块无数据时（如暂无任务完成记录、暂无社交好友），对应数据卡片/趋势图表/列表显示**空数据友好提示**（含图标+文字，如“暂无本周任务完成记录，快去任务模块完成任务吧！”+ 任务模块快捷入口），提示文字下方添加对应模块的快捷入口，一键跳转至该模块，避免用户无操作入口；当配置项修改后未生效时，显示“配置生效失败，点击重试”的提示，点击重试可重新同步配置至对应模块。

## 四、代码实现要点
1. **数据聚合实时更新机制**：前端使用Stream监听模块3-6数据变化，宠物升级/任务完成/激励获取时自动刷新个人中心数据；后端使用WebSocket推送实时数据更新，确保个人中心数据与各模块数据同步。
2. **配置实时同步机制**：前端使用Provider管理全局配置状态，配置修改后立即更新所有模块UI；后端配置字段存储为JSON格式，查询时动态过滤配置数据，配置修改后通过WebSocket推送至各模块。
3. **数据报表生成优化**：后端使用ExcelJS库生成Excel报表，使用csv-writer库生成CSV报表，报表数据从statistics表聚合查询，支持按模块/时间筛选；前端使用file_picker库选择保存路径，支持本地保存和云端备份。
4. **待办事项智能聚合**：后端定时任务（每1分钟）扫描各模块待办事项，聚合至user_center表；前端实时监听待办事项变化，自动更新待办列表和红点提醒。
5. **操作记忆持久化**：前端使用shared_preferences存储用户操作偏好（如趋势图表默认筛选模块、数据导出的默认格式/时间），退出APP后重新进入时恢复至上次操作状态；优化版同步至云端，跨设备保持一致操作体验。

## 六、核心业务流程
1. **数据查看流程**：用户进入个人中心→查看核心数据卡片→点击卡片跳转对应模块详情页→查看详细数据→返回个人中心
2. **待办处理流程**：用户点击待办事项入口→查看待办列表→点击"立即处理"跳转对应模块→完成待办事项→返回个人中心→待办列表自动刷新
3. **配置修改流程**：用户进入设置页→选择配置分类→修改配置项→点击保存/自动生效→配置同步至所有模块→显示"设置已生效"提示
4. **数据导出流程**：用户点击数据导出入口→选择筛选条件（模块/时间/格式）→点击导出→等待导出完成→选择保存路径→查看导出记录

## 七、模块联动关系
1. **与模块3（宠物成长）联动**：

   - 个人中心展示任务统计数据（累计完成数、今日完成数、任务难度分布）
   - 任务完成时，个人中心数据实时更新
   - 任务模块待办事项（如未完成适配任务）聚合至个人中心待办列表
3. **与模块5（激励获取）联动**：
   - 个人中心展示激励收益数据（当前积分、宝箱数量、成就解锁数）
   - 激励获取时，个人中心数据实时更新
   - 激励模块待办事项（如未领取激励、即将过期积分）聚合至个人中心待办列表
4. **与模块6（社交互动）联动**：
   - 个人中心展示社交数据（好友数、社交互动数、排行榜排名）
   - 社交互动时，个人中心数据实时更新
   - 社交模块待办事项（如未进行社交互动）聚合至个人中心待办列表

## 三、技术实现核心要求（前后端分工）
前端负责宠物进阶养成页面展示、图鉴卡片渲染、养成操作交互、图鉴解锁动画、本地养成进度存储，以及与模块3-7的快捷跳转联动；后端负责养成进度核心接口、图鉴解锁校验接口、养成收益反哺计算接口、道具使用校验接口，所有接口通过`versionType`区分版本，优化版需token令牌校验，接口返回格式统一为`{code: 状态码, data: 响应数据, msg: 提示信息}`。

### （一）数据库设计（新增+修改表，适配进阶养成+图鉴，基于模块3 pet表）
基于原有SQLite数据库，**新增`pet_advance`表（进阶养成核心表）、`pet_album`表（图鉴核心表）**，修改模块3`pet`表、模块4`task`表、模块5`incentive`表新增养成联动字段，所有字段量化可配置，确保与模块3-7数据互通。
1.  **pet_advance表**（新增，双版本，宠物进阶养成核心表）：存储属性、技能、皮肤的养成进度
    - `userId`：用户ID；`petId`：宠物ID（关联模块3 pet表）；`physical`：体力值；`intelligence`：智力值；`charm`：魅力值（三大属性，均为0-100）；
    - `skillLevel`：技能等级（JSON格式，如{skill1:3, skill2:5, skill3:2}，0-5级）；`skinList`：已收集皮肤列表（JSON格式，含皮肤类型、状态）；
    - `propBag`：养成道具背包（JSON格式，含属性丹、技能书、皮肤碎片的数量）；`advanceRate`：进阶养成总进度（0-100%）；
    - `updateTime`：养成进度更新时间。
2.  **pet_album表**（新增，双版本，图鉴核心表）：存储图鉴解锁状态、奖励领取记录
    - `userId`：用户ID；`petId`：宠物ID；`albumType`：图鉴分类（形象/技能/皮肤/成就）；`albumList`：子图鉴列表（JSON格式，含图鉴名称、解锁状态、解锁时间、奖励领取状态）；
    - `totalUnlock`：已解锁图鉴总数；`albumReward`：图鉴收集阶梯奖励领取记录（JSON格式）；`albumSign`：图鉴成就标识（如图鉴大师、皮肤收藏家）。
3.  **pet表**（修改，模块3核心表）：新增进阶养成联动字段
    - `advanceAddRate`：进阶养成收益加成比例（FLOAT，默认1.0）；`expReduceRate`：经验减少比例（FLOAT，默认1.0，属性值提升后降低）。
4.  **task表**（修改，模块4核心表）、**incentive表**（修改，模块5核心表）：新增养成道具掉落字段，标注各任务/激励可掉落的养成道具类型、数量。

### （二）后端（Node.js）开发：核心接口实现（新增+联动，对接模块3-7）
所有接口适配双版本，深度联动模块3基础养成接口、模块4任务收益接口、模块5激励道具接口，实现养成道具掉落、进度更新、收益反哺、图鉴解锁校验。
1.  **宠物进阶养成核心接口（新增，双版本通用）**
    接口地址：GET /api/pet/advance/core
    接收参数：userId、petId、versionType、token（优化版必填）
    核心逻辑：① 参数/令牌校验；② 从pet_advance表查询属性、技能、皮肤养成进度+道具背包；③ 从pet表查询进阶收益加成比例；④ 校验是否有可使用的道具、可解锁的养成等级；⑤ 返回养成进度+道具信息+可操作标识。
    返回数据：code=200，data={attrInfo: 属性信息, skillInfo: 技能信息, skinInfo: 皮肤信息, propBag: 道具背包, addRate: 收益加成比例}，msg="宠物进阶数据查询成功"。
2.  **养成道具操作接口（新增，双版本通用，含使用/合成子接口）**
    - 道具使用：POST /api/pet/advance/prop/use → 校验道具是否充足、养成等级是否达标，使用道具并更新养成进度，计算收益加成/经验减少比例，联动更新pet表；
    - 皮肤合成：POST /api/pet/advance/skin/synthesize → 校验皮肤碎片是否充足，合成完整皮肤并添加至skinList，触发皮肤收集提醒。
3.  **图鉴核心接口（新增，双版本通用，含查询/解锁/奖励领取子接口）**
    - 图鉴查询：GET /api/pet/album/query → 从pet_album表查询所有图鉴解锁状态、收集总数、阶梯奖励领取记录；
    - 图鉴解锁：POST /api/pet/album/unlock → 校验图鉴解锁条件（模块3-6数据），解锁图鉴并更新状态，触发解锁动画+奖励发放，联动模块5激励接口发放奖励；
    - 奖励领取：POST /api/pet/album/reward/receive → 校验图鉴收集数量，发放阶梯奖励（道具/收益加成/宝箱），更新奖励领取记录。
4.  **养成收益反哺接口（新增，双版本通用）**
    接口地址：POST /api/pet/advance/rate/link
    接收参数：userId、petId、advanceRate、versionType、token（优化版必填）
    核心逻辑：① 养成进度更新后自动调用；② 根据养成进度计算模块4-6的收益加成比例；③ 联动更新模块4 task表、模块5 incentive表的收益系数；④ 返回加成后的收益比例，实时生效。

### （三）前端（Flutter）开发：核心交互逻辑（新增+优化，适配单手握持+联动模块3-7）
#### 2. 核心交互逻辑（新增+优化，适配按键优化+评估系统）

- 按键视觉与状态分级：① 道具使用/技能升级/奖励领取等主养成按键，复用全局主按钮样式（#367BF5），尺寸为页面宽度75%/高度48px或圆形按键（50×50px），文字/图标16sp，**不可操作时置灰（#C0C4CC）+ 透明度60%**，添加悬浮提示标注不可操作原因（如“体力丹不足：当前0/1”“技能1未满级，无法升级技能2”“图鉴未解锁，无法领取奖励”）；② 皮肤穿戴/选择项采用“皮肤预览图+单选按钮”设计，选中的皮肤添加浅蓝边框（#1890FF）+“已穿戴”标识，未收集的皮肤为灰色模糊预览图，标注收集条件（如“需10个皮肤碎片合成”）；③ 图鉴卡片采用标准化卡片式设计，未解锁图鉴为灰色模糊状态，标注解锁条件（如“S级评估解锁”），已解锁图鉴为彩色清晰状态，添加“已解锁”金色标识，可点击查看图鉴详情。
- 点击反馈标准化统一：所有可点击按键触发**0.98倍缩放动效（0.2s）+ 轻量原生震动**（震动强度可在模块7设置中调节），核心养成按键（道具使用/技能升级/奖励领取）操作成功后，额外触发**彩色光效动效（0.3s）**（属性养成为绿色、技能升级为蓝色、皮肤合成为粉色、图鉴解锁为金色），并显示“操作成功”类浮窗提示（如“使用成功！体力+10”“技能1升级至2级”“皮肤合成成功”“图鉴解锁成功”），浮窗从按键位置向上漂浮至页面顶部后消失；图鉴卡片点击后触发淡入淡出动画，展示图鉴详情（含解锁条件、奖励、展示图）；皮肤穿戴点击后无延迟切换宠物形象，触发“皮肤穿戴成功”的轻量提示（1.5s自动消失），无需等待加载。
- 状态联动的按键权限：按键状态与**养成进度、道具数量、模块3-6数据**实时联动，无延迟响应，① 道具数量不足时，对应使用按键立即置灰，道具数量补充后（如从模块5领取激励），按键实时激活；② 养成等级未达标时，下一级升级按键置灰，等级达标后实时激活；③ 模块3-6数据未满足图鉴解锁条件时，图鉴卡片保持灰色模糊状态，条件达标后（如评估等级提升至S级），图鉴卡片自动解锁，触发解锁动画，按键实时激活可查看/领取奖励；④ 皮肤碎片集齐后，合成按键立即激活，添加小红点提醒，合成后按键恢复为未激活状态，皮肤预览图实时变为彩色清晰状态。

##### （2）宠物多线进阶养成交互逻辑（新增，核心衔接模块3/4/5）
- 养成进度实时可视化：宠物进阶页面按“属性养成、技能进阶、皮肤收集”三大维度分区，每区添加**进度总览卡片**，展示当前进度/满级进度（如“体力：25/100（25%）”“技能1：3/5（60%）”“皮肤收集：2/8（25%）”），进度采用**渐变进度条+数字**展示，使用道具/升级技能/合成皮肤后，进度条与数字实时更新，触发**进度条填充动画+数字渐变动画**，无需手动刷新；页面顶部添加“进阶养成总进度”卡片，展示三大维度的综合进度，总进度达到指定比例可解锁额外奖励（如总进度50%，解锁1个高级体力丹）。
- 养成道具一键操作：道具背包按“属性丹、技能书、皮肤碎片”分类展示，每个道具标注数量+使用/合成条件，支持**一键操作**：① 道具使用：选中道具后，可选择“使用1个/使用10个/一键使用至满级”，选择后点击“确认使用”，一键完成操作，无需逐个点击，使用后道具数量实时减少，养成进度实时提升；② 皮肤合成：皮肤碎片集齐后，合成按键添加小红点提醒，点击“一键合成”，直接合成完整皮肤，添加至皮肤列表，碎片数量清零，合成后触发皮肤收集提醒。
- 养成收益实时反哺：养成进度提升后（如属性值增加、技能升级），**收益加成比例/经验减少比例实时计算并同步至模块3-6**，同步完成后触发横幅提示（如“体力+10，宠物升级经验减少5%”“技能1升级至2级，任务收益+5%”），模块3-6的收益实时按新比例计算，无需重新进入模块；收益加成比例在进阶页面、模块3宠物主页、模块4任务页面、模块5激励页面均实时展示，标注“宠物进阶加成：+X%”。
- 养成提醒与快捷跳转：当背包中有可使用的道具、有可升级的技能、有可合成的皮肤时，宠物进阶页面/模块3宠物主页/模块7个人中心添加**小红点提醒**，点击提醒可一键跳转至对应养成页面；同时，养成所需道具的获取入口，在对应养成区标注（如“体力丹：模块5精英宝箱获取”“技能书：模块4高质量任务获取”），点击获取入口可一键跳转至模块4/5对应页面，简化道具获取流程。

##### （3）皮肤收集与穿戴交互逻辑（新增，衔接模块5/6）
- 皮肤分类展示与预览：皮肤页面按“基础皮肤、稀有皮肤、专属皮肤”三大类分区展示，每类皮肤按获取难度排序，皮肤预览图为宠物穿戴后的实际效果，支持**双指缩放/单指滑动**查看宠物全身形象，未收集的皮肤标注**详细收集条件**（如“基础皮肤1：模块5普通宝箱掉落，掉落概率30%”“稀有皮肤1：需10个皮肤碎片合成，碎片从模块5每日福利获取”“专属皮肤1：模块6排行榜今日榜一奖励”），收集条件标注关联模块入口，点击可一键跳转。
- 皮肤碎片实时合成：皮肤碎片在道具背包中按皮肤类型分类，每个碎片标注所属皮肤，碎片数量达到合成要求后，**合成按键自动激活并添加小红点提醒**，点击“一键合成”，碎片数量实时清零，皮肤立即添加至皮肤列表，预览图从灰色模糊变为彩色清晰，触发“皮肤合成成功”的动画+提示，同时皮肤穿戴按键激活，支持一键穿戴。
- 皮肤穿戴一键切换与预览：已收集的皮肤支持**一键穿戴/切换**，点击皮肤预览图下方的“穿戴”按键，宠物形象立即切换为该皮肤，页面中央展示宠物穿戴后的动态预览图（3秒），同时触发“皮肤穿戴成功”的提示；支持“皮肤预览”功能，未穿戴的皮肤可点击“预览”，查看宠物穿戴后的效果，预览不影响当前穿戴状态，预览完成后点击“关闭”返回原状态。
- 皮肤收集与图鉴联动：收集指定数量/类型的皮肤后，**自动触发皮肤图鉴解锁校验**，满足解锁条件后，皮肤图鉴立即解锁，触发金色光效的图鉴解锁动画，同时发放图鉴解锁奖励（如积分、高级皮肤碎片），奖励直接发放至模块5激励背包，触发“图鉴解锁成功！获得XX奖励”的弹窗提示，皮肤收集数量实时同步至图鉴页面，图鉴进度实时更新。

##### （4）宠物图鉴收集与解锁交互逻辑（新增，核心衔接模块3-6）
- 图鉴分类与快捷筛选：图鉴页面按“宠物形象、技能、皮肤、成就”四大类分区，顶部设置**分类筛选按键**（标签式设计），点击标签无延迟切换图鉴分类，筛选后图鉴列表实时刷新，未解锁图鉴与已解锁图鉴分开排列（已解锁在前、未解锁在后），支持按“解锁状态/收集时间”排序，满足用户不同查看需求。
- 图鉴解锁自动校验与动画：图鉴解锁条件与**模块3-6核心数据**实时联动，系统每秒自动校验一次，当模块3-6数据满足解锁条件时（如评估等级提升至S级、技能全满级、收集全皮肤、解锁全核心成就），**立即触发图鉴解锁流程**，首先弹出图鉴解锁提示（“恭喜！解锁XX图鉴”），随后触发**专属金色星光环绕动画**，图鉴卡片从页面底部弹出并放大展示3秒，展示完成后缩小至图鉴列表，状态从灰色模糊变为彩色清晰，添加“已解锁”金色标识，同时自动发放图鉴解锁奖励，奖励直接到账并触发提示。
- 图鉴详情与奖励领取：已解锁图鉴卡片可点击查看详情，详情页包含**图鉴名称、解锁时间、解锁条件、解锁奖励、高清展示图**，以及“设置为展示图鉴”按键，点击该按键，图鉴可在模块7个人中心、模块6社交排行榜展示，添加专属图鉴标识；图鉴解锁奖励**自动发放**，无需手动领取，若奖励为养成道具/积分，直接添加至对应背包，若奖励为收益加成，实时计算并同步至模块3-6，奖励发放后在详情页标注“奖励已发放”。
- 图鉴收集阶梯奖励交互：图鉴页面底部添加**收集阶梯奖励卡片**，展示当前收集数量/阶梯数量（如“当前收集12/20，解锁进阶宝箱”），进度采用渐变进度条展示，收集数量达到阶梯要求后，奖励卡片添加小红点提醒，点击“领取奖励”按键，一键领取阶梯奖励（如稀有养成道具、永久收益加成、模块5专属宝箱），领取成功后触发动画+提示，奖励直接到账，进度条实时更新至下一阶梯。

##### （5）跨模块联动与养成闭环交互逻辑（新增，衔接模块3-7）
- 养成道具获取快捷跳转：在宠物进阶页面的每个养成维度，均标注**道具获取入口**（如“体力丹：模块5精英宝箱”“技能书：模块4高质量任务”“皮肤碎片：模块6好友助力”），获取入口为可点击的文字链接，点击后**一键跳转至对应模块的道具获取页面**，如点击“模块5精英宝箱”，直达模块5宝箱开启页面，点击“模块4高质量任务”，直达模块4高难度任务推荐页面，简化道具获取流程，推动用户完成模块4-6的核心操作。

##### （6）全局异常处理与通用交互逻辑（优化，统一全模块体验）
- 网络异常交互：网络断开时，宠物进阶/图鉴页自动切换至本地模式，顶部显示黄色提示栏（“网络异常，仅可查看本地养成进度，无法使用道具/合成皮肤/解锁图鉴”），本地可正常查看养成进度、道具数量、图鉴状态，无法进行任何养成操作；网络恢复后，提示栏自动消失，**异步同步本地未同步的养成数据至云端**，同步完成后刷新页面，补充执行网络异常期间未完成的操作（如道具使用、皮肤合成），并触发“网络恢复，数据已同步”的提示。
- 操作错误与异常提示：① 道具数量不足时，点击使用按键触发toast提示（如“体力丹不足，当前0/1，快去模块5获取”）；② 养成等级未达标时，点击升级按键触发提示（如“技能1需5级满级，当前3级，无法升级”）；③ 图鉴未解锁时，点击领取奖励按键触发提示（如“图鉴未解锁，需完成S级评估解锁”）；④ 服务器错误导致操作失败（如道具使用、皮肤合成、图鉴解锁），弹出弹窗提示（“操作失败，请点击重试”），含“重试/取消”按键，点击重试可重新调用接口，不影响其他操作。
- 加载交互标准化：页面级操作（进入宠物进阶页、图鉴页、皮肤详情页）显示**全屏轻量加载**（含宠物图标+“加载养成数据中...”文案），按钮级操作（道具使用/合成、技能升级、奖励领取、图鉴解锁）显示**按钮内小型加载动画**（不改变按钮尺寸/位置），无重复加载、过度加载；养成进度/图鉴状态更新时，采用**局部刷新**，仅刷新对应模块，不刷新整个页面，提升加载体验。
- 空数据与异常状态处理：当背包中无养成道具、无已收集皮肤、无已解锁图鉴时，对应页面显示**空数据友好提示**（含可爱宠物图标+文字+快捷入口），如“暂无养成道具，快去模块5开启宝箱获取吧！”+ 模块5快捷入口，“暂无已收集皮肤，快去合成皮肤吧！”+ 皮肤合成页面入口，“暂无已解锁图鉴，快去完成模块3-6任务解锁吧！”+ 模块3评估页面入口，避免用户无操作入口，同时引导用户前往对应模块获取/解锁。
- 操作记忆与个性化：系统自动记忆用户的**操作偏好**，如图鉴页面的默认筛选分类、皮肤页面的默认展示类型、进阶页面的默认展开分区，退出APP后重新进入时，恢复至上次操作状态，无需重复设置；偏好数据存储在`user_center`表（模块7），基础版本地保存，优化版云端同步，跨设备保持一致操作体验；同时支持“恢复养成操作记忆”，一键重置为系统默认偏好。
