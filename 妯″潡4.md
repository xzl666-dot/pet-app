# 模块四：学习任务模块（详细版）
本模块是用户登录后的默认首页，也是网站的核心功能载体，承担“任务驱动学习、双核心元素联动激励”的核心职责。整体界面采用“功能分区明确、视觉层次清晰”的设计思路，以浅蓝色为主色调，搭配柔和的白色背景，营造出专注且轻松的学习氛围。模块围绕“任务完成-奖励获取-宠物养成-等级提升”的正向循环设计，所有功能均服务于提升用户学习积极性的目标，整体篇幅约3000字。

## 一、模块整体设计原则
1.  **功能联动性原则**
    本模块是动物养成模块和价值评估中心模块的核心联动枢纽，所有任务操作均直接影响双核心数据。任务完成后的奖励与宠物属性、评估等级强绑定，宠物状态反过来影响任务奖励倍率，评估等级决定任务难度和类型，形成闭环激励体系。数据同步采用“实时更新+页面即时刷新”的机制，确保用户操作后能立即看到反馈，增强参与感和成就感。

2.  **视觉聚焦性原则**
    页面采用“左辅右主”的布局结构，左侧宠物养成区作为辅助激励元素，右侧任务核心区作为用户主要操作区域，视觉重心偏向右侧，引导用户聚焦学习任务。界面元素按“重要性层级”区分样式：核心操作按钮（完成任务、投喂）采用高饱和度的蓝色，次要元素（标签、筛选）采用浅灰色，文字按标题、内容、提示分为三级字号和颜色，避免视觉混乱。

3.  **操作轻量化原则**
    任务操作流程控制在2步以内，用户完成线下学习后，仅需“点击任务→确认完成”即可获取奖励，无需复杂步骤。页面支持“一键刷新”“一键筛选”等快捷操作，历史任务查询支持按日期快速定位，减少用户操作成本。所有交互反馈均采用简洁的动画和弹窗，例如宠物升级时的弹窗提示、任务完成时的奖励动画，时长控制在1秒内，不干扰用户后续操作。

4.  **个性化适配原则**
    任务列表根据用户当前评估等级动态匹配，不同等级用户看到的任务难度、数量、奖励完全不同，实现“因材施教”的个性化适配。页面支持“任务偏好设置”，用户可选择擅长的学科类型，系统优先展示对应学科任务，提升任务完成率。宠物形象和状态实时展示，强化用户的专属感和陪伴感。

## 二、页面整体布局设计
页面采用固定宽度的居中布局，最大宽度1200px，左右边距自适应，确保在不同分辨率下布局稳定。整体分为三大区域，按从上到下的顺序排列，区域间采用浅灰色分隔线区分，层次分明。

1.  **顶部状态栏（通栏设计，高度60px）**
    状态栏为固定定位，始终悬浮在页面顶部，不随滚动条移动，方便用户随时查看核心信息和操作。
    - **左侧区域**：用户头像+昵称，头像为圆形（直径40px），支持点击进入系统设置模块；昵称字体大小16px，颜色#333333，加粗显示。
    - **中间区域**：当前评估等级展示，采用“等级图标+文字+进度条”的组合样式。等级图标为卡通勋章样式，例如D级为灰色勋章，S级为金色勋章；文字显示“当前等级：B级”，字体大小14px；进度条为蓝色渐变样式，显示当前分数距离下一等级的进度，进度条右侧标注“当前分数：120/200”，字体大小12px，颜色#666666。点击该区域可直接跳转至价值评估中心模块。
    - **右侧区域**：两个功能按钮，分别是“刷新任务”和“退出登录”。“刷新任务”按钮为蓝色文字+刷新图标，点击后可手动刷新当前任务列表（每日仅可手动刷新1次）；“退出登录”按钮为红色文字，点击后弹出确认弹窗，确认后清除登录状态，跳转至登录页。

2.  **左侧宠物养成区（宽度300px，高度600px，固定定位）**
    位于页面左侧，宽度固定，高度自适应，与右侧任务核心区并列排布。区域背景为白色，带有轻微的圆角和阴影，营造出卡片式的悬浮效果。
    - **顶部宠物形象展示区（高度300px）**：占据区域上半部分，背景为浅蓝色渐变，中央展示用户的宠物动态形象。宠物形象根据等级不同而变化，例如LV.1为小猫形象，LV.5为大猫形象，等级越高形象越精致。宠物会根据状态做出不同动作，例如营养值充足时会跳跃，营养值不足时会低头，增强交互趣味性。形象下方显示宠物等级“LV.3”，字体大小20px，加粗显示。
    - **中间属性面板（高度200px）**：采用四宫格布局，分别展示宠物的核心属性：经验值、营养值、亲密度、拥有饲料。每个属性均采用“图标+名称+数值”的样式，例如经验值图标为书本，显示“经验值：80/100”；营养值图标为爱心，显示“营养值：75/100”，并以进度条形式展示数值占比；亲密度图标为星星，显示“亲密度：60/100”；拥有饲料图标为饲料袋，显示“饲料：12个”。所有文字字体大小12px，颜色#666666。
    - **底部操作按钮区（高度100px）**：包含一个“投喂饲料”按钮，按钮宽度占区域的80%，高度40px，背景色为橙色，文字颜色白色，字体大小14px。按钮禁用条件为“拥有饲料数量为0”，禁用时背景色为浅灰色，无法点击。点击按钮后，消耗1个饲料，营养值增加5点，同时宠物形象做出进食动作，弹窗提示“投喂成功！营养值+5”。

3.  **右侧任务核心区（宽度850px，与左侧宠物区间距50px）**
    位于页面右侧，是用户的核心操作区域，占据页面主要视觉空间。区域背景为白色，顶部为标签栏，下方为任务列表，支持滚动加载更多任务。
    - **顶部标签栏（高度50px）**：包含三个标签，分别是“今日任务”“进阶任务”“历史任务”，标签采用横向排列，宽度均等。标签默认选中“今日任务”，选中状态下底部显示蓝色下划线，文字颜色#1677ff，加粗显示；未选中状态下文字颜色#999999。点击标签可切换至对应任务列表，切换时采用淡入淡出动画，时长300ms。
    - **任务筛选区（高度40px）**：位于标签栏下方，包含“学科筛选”下拉框和“难度筛选”下拉框，两个下拉框横向排列，间距20px。学科筛选包含“全部、语文、数学、英语、物理、化学、生物、历史、地理、政治”选项；难度筛选包含“全部、简单、中等、困难”选项。筛选条件实时生效，点击下拉框选项后，任务列表立即刷新为符合条件的任务。
    - **任务列表区（高度自适应）**：采用卡片式布局，每个任务为一个独立卡片，宽度100%，高度120px，卡片之间间距15px，带有轻微阴影和圆角。任务卡片包含以下元素：
      - 左上角：任务类型标签，例如“基础任务”“语文任务”，背景色为浅蓝色，文字颜色白色，字体大小12px，圆角设计。
      - 中上部分：任务标题，字体大小16px，颜色#333333，加粗显示，例如“完成数学第一章课后习题”。
      - 中部：任务描述，字体大小12px，颜色#666666，例如“完成习题1-10题，要求正确率不低于80%”。
      - 右下角：奖励信息+操作按钮，奖励信息显示“宠物经验+10、评估分数+5、饲料+1”，字体大小12px，颜色#666666；操作按钮为“完成任务”，宽度100px，高度30px，背景色#1677ff，文字颜色白色，未完成任务时为可点击状态，完成后变为“已完成”灰色状态，无法点击。

## 三、核心功能与使用流程
本模块的核心功能围绕“任务管理”“双核心联动”展开，所有功能流程均设计得简洁易懂，确保不同年龄段的用户都能快速上手。

1.  **任务匹配与展示功能**
    - **动态匹配机制**：用户登录后，系统根据当前评估等级自动匹配任务列表。D级用户仅展示基础任务，数量为5个/天；C级用户展示基础任务+少量简单进阶任务，数量为6个/天；B级及以上用户展示全部类型任务，数量为8个/天。任务难度随等级提升而增加，奖励也相应提高，例如D级任务奖励宠物经验+5，S级任务奖励宠物经验+20。
    - **个性化偏好设置**：用户可点击任务筛选区的“偏好设置”按钮（隐藏在筛选区右侧，鼠标悬浮时显示），选择3个擅长的学科，系统后续会优先展示对应学科的任务，占比提升至50%。偏好设置保存后长期有效，用户可随时修改。
    - **任务刷新规则**：任务列表每日凌晨0点自动刷新，已完成任务归档至“历史任务”标签，未完成任务自动清零。用户每日可手动点击“刷新任务”按钮1次，刷新当前任务列表，替换掉不喜欢的任务，手动刷新后的任务难度与原任务一致。

2.  **任务完成与奖励领取功能**
    这是本模块的核心操作流程，用户完成线下学习后，通过该功能获取奖励，驱动双核心数据更新。
    - **操作流程**：
      1.  用户在任务列表中找到已完成的线下任务，点击任务卡片右下角的“完成任务”按钮。
      2.  系统弹出确认弹窗，显示“请确认已完成该任务，确认后将获得奖励”，包含“确认”和“取消”两个按钮。
      3.  用户点击“确认”后，弹窗关闭，按钮状态变为“已完成”，同时触发奖励动画，任务卡片右侧弹出奖励提示框，显示“奖励已发放：宠物经验+10、评估分数+5、饲料+1”。
      4.  系统实时更新双核心数据：宠物经验值增加对应数值，若经验值达到升级阈值，宠物自动升级；评估分数增加对应数值，若分数达到等级提升阈值，评估等级自动提升；饲料数量增加对应数值。
    - **奖励倍率调整机制**：宠物营养值直接影响任务奖励倍率，当营养值≥80时，奖励倍率提升至120%；当营养值≤50时，奖励倍率降低至80%；营养值在50-80之间时，奖励倍率为100%。例如营养值为90时，原本奖励宠物经验+10，实际获得12点，该机制激励用户关注宠物状态，保持宠物营养充足。
    - **异常处理**：若用户重复点击已完成任务的按钮，系统弹出提示“该任务已完成，无法重复领取奖励”；若网络中断导致奖励未发放，用户重新登录后，系统会自动补发未领取的奖励，并弹窗提示。

3.  **历史任务查询功能**
    用户可通过该功能查看过往完成的任务，回顾学习轨迹，增强学习的成就感。
    - **查询方式**：用户点击“历史任务”标签，切换至历史任务列表，列表默认按“完成时间倒序”排列，最新完成的任务显示在最上方。
    - **筛选功能**：历史任务列表支持按“日期范围”“学科类型”“任务难度”筛选，用户可选择“近7天”“近30天”“全部时间”等日期范围，快速定位特定时间段的任务。
    - **任务详情**：点击历史任务卡片，可查看任务的详细信息，包括完成时间、获得的奖励、当时的评估等级等，详情弹窗支持截图保存，方便用户分享学习成果。

4.  **双核心数据联动反馈功能**
    本模块与动物养成模块、价值评估中心模块的联动反馈是激励用户的关键，所有数据更新均有明确的视觉反馈。
    - **宠物养成联动反馈**：
      1.  任务完成获得经验值后，宠物属性面板的经验值进度条实时增长，进度条颜色随数值增加逐渐变深。
      2.  经验值达到升级阈值时，页面弹出“宠物升级啦！”的弹窗，显示宠物升级前后的形象对比，同时提示“升级成功！亲密度+10”，弹窗自动关闭后，宠物形象更新为新等级的形象。
      3.  投喂饲料后，营养值进度条实时增长，宠物形象做出对应动作，例如营养值从40提升至60时，宠物从低头状态变为跳跃状态。
    - **价值评估联动反馈**：
      1.  任务完成获得分数后，顶部状态栏的进度条实时增长，分数数值同步更新。
      2.  分数达到等级提升阈值时，页面弹出“等级提升啦！”的弹窗，显示等级提升前后的勋章对比，同时提示“恭喜升级为B级！解锁进阶任务权限”，弹窗自动关闭后，任务列表自动新增进阶任务。
      3.  点击顶部状态栏的评估等级区域，可快速跳转至价值评估中心模块，查看详细的分数变化历史和等级权益。

## 四、跳转逻辑设计
本模块作为核心枢纽，与其他模块的跳转逻辑设计得清晰便捷，确保用户在不同模块间切换流畅。

1.  **模块内跳转**
    - 点击顶部状态栏的评估等级区域 → 跳转至**模块六 价值评估中心模块**。
    - 点击左侧宠物养成区的宠物形象 → 跳转至**模块五 动物养成模块**。
    - 点击任务标签栏的“今日任务”“进阶任务”“历史任务” → 切换至对应任务列表，无页面跳转。
    - 点击任务筛选区的选项 → 实时刷新任务列表，无页面跳转。

2.  **跨模块跳转**
    - 点击顶部状态栏的用户头像 → 跳转至**模块八 系统设置模块**。
    - 点击顶部状态栏的“退出登录”按钮 → 确认后跳转至**模块三 登录页**，清除所有登录状态和缓存数据。
    - 点击历史任务卡片的“查看详情” → 弹出详情弹窗，无页面跳转，弹窗支持点击右上角关闭按钮关闭。

3.  **外部跳转**
    任务描述中若包含学习资料链接（如电子课本、习题解析），点击链接后在新标签页打开，不影响当前模块的操作。

## 五、通用交互与异常处理规则
1.  **通用交互规则**
    - 页面滚动时，顶部状态栏固定悬浮，左侧宠物养成区随滚动条移动，右侧任务核心区正常滚动，确保用户随时能查看核心信息和操作宠物。
    - 所有按钮均添加hover效果，鼠标悬浮时颜色加深5%，光标变为手型，明确提示可点击。
    - 任务完成、宠物升级、等级提升等关键操作均有简洁的动画反馈，动画时长控制在1秒内，避免干扰用户。
    - 页面支持“一键返回顶部”功能，当滚动条距离顶部超过300px时，右下角弹出返回顶部按钮，点击后快速回到页面顶部。

2.  **异常处理规则**
    - 网络中断时，页面弹出“网络异常，请检查后重试”的提示框，包含“重试”按钮，点击后重新加载页面数据，无需用户重新登录。
    - 数据加载失败时，任务列表区显示“数据加载失败，请点击重试”的提示文字，点击后重新发起请求。
    - 用户长时间未操作（超过30分钟），系统自动弹出“您已长时间未操作，是否继续”的提示框，5分钟内未响应则自动退出登录，跳转至登录页，确保账号安全。
    - 任务奖励发放失败时，系统自动记录未发放的奖励，用户重新登录后自动补发，并弹窗提示“补发奖励已发放”。
